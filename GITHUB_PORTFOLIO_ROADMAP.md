Primary themes:

## Near-Term GitHub Actions

1. Pin `OptionPricer` as the polished C++ quant engineering project.
2. Pin QuantLab as the active flagship, clearly labeled in progress.
3. Keep placeholder modules private or unpinned until they have executable demos.
4. Avoid pinning coursework-style, step-only, or notebook-only repos.
5. Convert future research projects into focused repos only when each has a reproducible demo, README, and research memo.

## Repository Status Rules

- **Demo-ready**: has a clean README, reproducible build/run path, sample outputs, and no local artifacts.
- **In progress**: active implementation with a clear roadmap and implemented core components.
- **Planned/TBD**: concept exists, but should stay private or unpinned until there is a working demo.
- **Archive/private**: old coursework, duplicated step repos, placeholders, or repos without a public research narrative.
